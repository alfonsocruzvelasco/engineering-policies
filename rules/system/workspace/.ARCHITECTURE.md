# Workspace Storage Architecture
Author: Alfonso Cruz
Updated: Sun Dec  7 12:24:43 PM CET 2025

## Root Policy
/workspace is primary working storage, RAID resilient.

## Symlink Mapping
~/docker    → /workspace/containers
~/datasets  → /workspace/datasets
~/ai        → /workspace/ai

## Access Rules
Root owns workspace.
Developers operate via group permissions + docker group.

## Usage Rules
- No compiled artifacts stored here unless cached
- Only source, data, container infra and ML models
- Persistent DB data should live in Docker volumes
